## Design

### 1. Domain services + command metadata (#458)

Layout:

```text
src/core/
  vocab.rs          (from #460)
  projection.rs     (project/activate/orphan flows)
  lifecycle.rs      (commit/release/rollback/diff/history)
  inventory.rs      (read models, scoring)
  mod.rs
```

Rules:

1. Services take typed inputs (core structs, not clap types) and return typed
   results; no `serde_json::Value` shaping and no envelope construction
   inside `core/`.
2. `cmd_*` becomes: parse -> call service -> wrap envelope. Handlers move
   toward the 200-400 line norm as logic drains into services.
3. Command metadata: one declarative table adjacent to the `Command` enum -
   `fn command_meta(cmd: &Command) -> CommandMeta { records_audit,
   durable_audit, is_preview }` generated by a single match or macro, deleting
   `command_records_audit` and `command_requires_durable_audit` as separate
   trees. Exhaustiveness keeps the compiler enforcing coverage.
4. Panel mutation routes call services directly but keep
   `ensure_mutation_authorized`, lock acquisition, audit append, and envelope
   parity; the ADR section 4 route table stays the contract and its parity
   tests keep passing. Migration is per-route, starting with projection and
   lifecycle routes.

### 2. Single operation-log authority (#459)

1. Spec the journal-backed replacements for the pending-queue semantics that
   `sync push/pull/replay`, `ops retry/purge`, and `ops history repair`
   rely on, including the `loom-history` branch interplay - this is the risk
   center and gets its own review round before code.
2. Migrate command families one at a time behind the existing CLI surface;
   each migration PR includes an equivalence test against recorded
   pending-queue fixtures.
3. Delete `pending_ops.*` writers and files; one release note; no dual-write
   period longer than one release (V1 no-back-compat policy).
4. Update `docs/LOOM_ARCHITECTURE_DECISIONS.md` section 1 and
   `docs/LOOM_STATE_MIGRATION_NOTES.md`.

### 3. Shared vocab enums (#460)

```rust
// src/core/vocab.rs (ts_rs-exported like state_model types)
pub enum AgentKind { Claude, Codex, Cursor, Windsurf, Cline, Copilot,
                     Aider, Opencode, GeminiCli, Goose }
pub enum Ownership { Managed, Observed, External }
pub enum ProjectionMethod { Symlink, Copy, Materialize }
pub enum Health { Healthy, Drifted, Missing, Conflict, Orphaned }
pub enum MatcherKind { PathPrefix, ExactPath, Name }
```

1. Serde `rename_all` matches current on-disk spellings - existing files load
   unchanged, no data migration.
2. `state_model` structs replace `String` fields with these enums;
   `src/cli/agent_kind.rs` is deleted and the CLI re-exports the core enum
   (ValueEnum derived where needed).
3. Unknown-value policy per ADR section 2.1: `ownership`, `method`, `health`,
   `kind` fail load with a typed SCHEMA error; `agent` keeps forward
   compatibility via a dedicated `AgentId(String)` newtype validated against
   `AgentKind` on write paths only - readers preserve unknown agents, writers
   cannot emit them. This honors the ADR's reader/writer asymmetry with types
   instead of convention.

### 4. Adapter metadata completion (#461)

1. Follow the codex pattern in `metadata.rs`: give claude real
   `discovery_roots` (user `~/.claude/skills`, project `.claude/skills`),
   visibility semantics, and reload semantics; then the remaining eight.
2. Capabilities derive from metadata (e.g. `reload_required` from reload
   semantics) instead of one shared literal.
3. `health_checks`: wire into `AgentAdapter`, emit in `adapters_json()`, and
   consume in `workspace doctor` as advisory checks - or delete the field
   from both record types and `docs/schemas/agent-adapter-v2.schema.json`.
   Default decision: delete unless a concrete consumer lands in the same PR;
   declared-but-unwired schema surface is worse than absent surface.

## Sequencing And Risk

1. #460 first (mechanical, high type-safety payoff), #461 in parallel.
2. #458 next, one domain at a time (projection -> lifecycle -> inventory);
   each extraction PR is behavior-neutral and keeps contract tests green.
3. #459 last, gated on its own reviewed migration plan.
4. GH452 command deletions and this epic touch the same dispatch code; land
   GH452 #454/#455 either clearly before or clearly after the #458
   extraction of the affected domain to avoid double rebases.
